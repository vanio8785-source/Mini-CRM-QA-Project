### 1. Introduction
This document describes the test strategy and scope for the Mini CRM Demo Web Application.

### 2. Objective
To verify that the system functions correctly according to defined requirements and to identify defects before release.

### 3. Scope

### In Scope:
- Login functionality
- Dashboard display
- Search functionality
- CRUD operations (Admin)
- Role-based access control

### Out of Scope:
- Performance testing
- Security penetration testing
- Mobile responsiveness testing

### 4. Test Types
- Functional Testing
- Negative Testing
- Boundary Value Testing
- Role-based Testing
- Regression Testing

### 5. Test Environment
- OS: Windows 11
- Browser: Google Chrome (latest version)
- Frontend: HTML, CSS, JavaScript
- Data Source: Local JavaScript data file

### 6. Entry Criteria
- Application deployed locally
- Test data available
- Requirements defined

### 7. Exit Criteria
- All test cases executed
- Critical and High defects resolved
- Test Summary Report completed

### 8. Deliverables
- Test Cases
- Bug Reports
- Test Summary Report
